## [v1.1] - 2026-03-07

### 新增（基于PreCode Architect思想）

#### 🔴 强制性规范
- **AI_CONDUCT.md** - AI行为规范
  - 禁止虚报任务完成
  - 禁止创建V2/增强版文件
  - 诚实报告进度（使用百分比）
  - AI天性约束机制
  
- **TDD_MANDATE.md** - TDD强制规范
  - 红-绿-重构循环强制要求
  - 新代码覆盖率100%
  - 整体覆盖率≥90%
  - 禁止修改测试假装通过
  - 禁止先写代码后补测试

- **CODE_REVIEW.md** - 代码审查规范
  - 三层防护网（IDE/预提交/CI）
  - 质量门禁详细配置
  - 架构合规检查（ArchUnit风格）
  - 偏离度报告机制

#### 🟡 推荐性规范
- **FILE_MANAGEMENT.md** - 文件管理规范
  - docs/ 文档分类结构
  - 文件命名规范
  - 禁止版本号文件名
  - 文档生命周期管理

### 更新
- **README.md** - 规范总览
  - 新增强制性规范列表
  - 新增P0/P1/P2优先级
  - 新增绝对禁止行为清单
  - 新增任务验收标准
  - 新增AI快速开始指南

### 变更动机
基于PreCode Architect —— VSCode IDE插件原子化开发需求文档的思想：
1. **AI天性约束**: AI倾向于尽快完成任务而非高质量完成，需要强制性规范约束
2. **TDD强制执行**: 防止AI偷工减料、事后补测试
3. **三层防护网**: IDE实时+预提交+CI门禁，确保代码质量
4. **文档分类**: 完善docs/目录结构，便于管理和查找

## 更新流程

1. **识别需求**: 根据实践经验或外部输入识别规范改进点
2. **讨论评审**: 与人类监督员讨论变更必要性
3. **直接修改**: 在原规范文件上修改（禁止创建V2版本）
4. **更新日志**: 在CHANGELOG.md记录变更
5. **更新版本**: 更新文档头部的版本号
6. **通知相关方**: 确保所有AI Agent和开发者知晓变更
