## 📖 术语表（统一使用）

为避免中英文混用造成混乱，本项目统一使用以下术语：

| 英文                          | 中文         | 说明                         |
| ----------------------------- | ------------ | ---------------------------- |
| Layer 0 / Core Kernel         | 核心内核层   | 系统的信任锚点，不可自动修改 |
| Layer 1 / Skills Library      | 技能库层     | 存储已验证的代码模板和策略   |
| Layer 2 / Application Factory | 应用工厂层   | 任务管理和多任务并发         |
| Generation                    | 进化代       | 一次代码生成+测试+评估的迭代 |
| Evolution                     | 进化         | 通过迭代改进代码的过程       |
| Strategy                      | 策略         | 生成代码的方法和参数配置     |
| Thompson Sampling             | 汤普森采样   | 多臂老虎机算法用于策略选择   |
| Goodhart                      | 古德哈特     | 度量指标变成目标后失效的现象 |
| Sandbox                       | 沙盒         | 隔离的执行环境               |
| TDD                           | 测试驱动开发 | 先写测试后写代码的开发方法   |
| AAA Pattern                   | AAA模式      | Arrange-Act-Assert 测试结构  |
| Code Coverage                 | 代码覆盖率   | 被测试覆盖的代码比例         |
| Branch Coverage               | 分支覆盖率   | if/else等分支的覆盖比例      |

> **命名一致性**: 架构文档中使用"Layer 0/1/2"或"核心内核层/技能库层/应用工厂层"均可，但同一文档内应保持一致。
